Project Test Plan
Goals
  • Visualize their course progression throughout their degree
  • Find information about possible minors
  • Identify which complementary courses are available (updated over time)
  • Compare programs and curriculums
  • Quickly access all degree course information with only a few clicks
Benchmark Tests
  • The user can access their degree progression within 30 seconds.
  • The user can find information about any possible minors for ECSE students within 1 minute.
  • The user can access their updated complementary courses list within 30 seconds.
  • The user should be able to compare programs and curriculums within 1.5 minutes.
  • The platform is structured for the user to quickly access all ECSE degree course information with only a few clicks

Observer Briefing

As an observer for today's user test session, you'll be tasked with recording, analyzing, and potentially helping the user as they navigate the McGill Course Tracker web application prototype according to a series of predefined tasks (see Tasks). You will follow a similar, if not the exact, course as written in the Test Script, and describe to the user their tasks with no more detail than what is given. You may not offer any additional assistance to the user or veer from this script unless it is clear that they are making no progress.

Please review the Data Collection document, which you will be filling out as you observe the user throughout this activity. There is a data collection document for each observed user. Note down any point which causes them difficulty and anything significant they say over the process. Add as many comments as you think necessary to sufficiently convey what the user has done. Ensure that the user has completed the pre-test questionnaire prior to starting the tasks and that they complete the post-test questionnaire afterwards.

Good luck!


User Introduction

You have agreed to be a test user in today's user test session for the McGill Course Tracker, a web application aimed towards McGill students, prospective students, and advisors to help track progress towards degree completion. You will be given a prototype to navigate according to a series of predefined tasks that will be explained to you by an observer with knowledge of how the system works. If you were unaware, please note that this prototype is fully functional, nor are you here to test its functionality. The prototype is meant to represent the user interface of the product, which you will be exploring. You will be recorded throughout the session, which will later be used for analytical purposes.

Feel free to ask the observer to clarify tasks if you have trouble understanding them, but do your best to refrain from asking questions about the system itself unless you are truly struggling. We encourage you to verbalize your thought process as you go through the prototype, but do not feel required to do so. Do not feel discouraged if you have trouble using the interface, as this exercise is meant to find things that can be improved about it.

Your input is valuable and we appreciate your time. Thank you, and good luck with the proceedings.

Training Documentation

This document is meant to explain to you, the user, all the information you will need to complete your tasks. During the test session, you are to evaluate the prototype provided to you by your observer/guide on our upcoming product, the McGill Course Tracker.

The McGill Course Tracker is a website meant to help current, incoming, and prospective students and academic advisors with course selection and tracking degree progression. Anyone can create an account on the website and use it for these purposes. The primary feature is a page where you can input courses that will then calculate your credits, needed credits, and both completed and incomplete requirements. Multiple course lists can be created and saved to your account to be used as reference.

Test Script

This document provides the general script to be delivered by you, the observer, to the user over the testing process. You do not need to follow this script exactly. Deviate as you deem necessary in order to progress through the tasks.

SCRIPT BEGIN

Hello! Welcome to the user testing session for the McGill Course Tracker website, meant to help current, prospective, and incoming students as well as advisors through course selection and determining degree progression. I'll be observing and guiding you through a set of tasks we have set out. You should know that you may withdraw consent for this study and exit at any time. If there is anything you're uncomfortable with, please let me know, and we will try to accommodate you. Please note that you will be recorded.

First of all, please confirm that you've completed the pre-test questionnaire. You should have been given it prior to today.

[Wait for confirmation.]
  • If completed: Excellent. Let's begin.
  • If not completed: I have a copy of the questionnaire with me here. You can take the next 10 minutes or so to fill it out.
[With the questionnaire confirmed to be complete, show the prototype.]

Today, youlll be evaluating one of these. This is a prototype user interface for the website. It is not completely functional, and I will be playing the part of acting out the system's expected behaviour. Feel free to ask questions if you're unclear on any of the tasks, but otherwise, you will go through each one without help, unless you're truly stuck.

We encourage you to verbalize your thoughts during this session, which will give us better insight into the user's experience.

Do you have any questions?

[Wait for response. If none, continue. If there are, answer if you think it's appropriate.]

Excellent. Let's begin.

There are 9 tasks planned for today. The first one is to create an account. Here's the homepage.

[User creates account.]

Your second task is to log out of the account you just created.

[User logs out of account.]

And third, now log back in.

[User logs back into account.]

Now, create a course list and input your courses.

[User creates course list.]

Fifth, now that the list is complete, use it to calculate your credits and credits to completion.

[User calculates credits.]

Now use the same list to get all your unfulfilled course requirements.

[User gets course requirements.]

Great. Now for task seven, save this course list to your account.

[User saves course list.]

For task eight, I want you to access the course list you just saved.

[User opens saved task list.]

And now, lastly, for task nine, edit this course list and save the changes.

[User edits the course list and saves.]

Great! You're done. I'll stop the recording now. Do you need a break?
  • If yes: [Give the user 5 minutes.]
  • If not: [Continue on to post-test questionnaire.]
I'd now like you to fill out this post-test questionnaire. Take as much time as you need, and ask me if you're confused about anything here.

[User completes post-test questionnaire.]

And that's it! You're done for today. Thank you so much for your time. We'll send you the details of the website once everything has been completed. Have a good rest of your day!

END SCRIPT
